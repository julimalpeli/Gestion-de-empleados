## ⚠️ PROBLEMAS IDENTIFICADOS EN EL CÁLCULO

### **PROBLEMA 1: Concepto de "Mejor Sueldo"**

El código actual usa:
```typescript
const totalPaid = depositoAmount + efectivoAmount;
const adjustedSalary = Math.max(totalPaid - excludedConcepts, 0);
```

**Pero el PDF muestra que está incluyendo:**
- ✅ Sueldo base
- ✅ Antigüedad (apropiado)
- ❓ Adicional Asistencia (49.239,45) - depende de si es habitual
- ❓ Complemento de Servicio (59.087,34) - depende de si es habitual
- ❓ Adicional Zonal 10% (49.239,45) - depende de si es habitual

**Recomendación:**
- Revisar con contador/abogado laboral cuáles conceptos deben incluirse
- En muchas jurisdicciones, solo el sueldo base + antigüedad son incluibles
- Los adicionales pueden no contarse para SAC si no son permanentes

### **PROBLEMA 2: Fallback a payrolls fuera del semestre**

Si no hay payrolls en el semestre 1-2026, el código cae a:
```typescript
const relevantPayrolls =
  payrollsInSemester.length > 0 ? payrollsInSemestre : employeePayrolls;
```

Esto significa que podría usar salarios de **2025 o antes** para calcular el aguinaldo de 2026, lo cual es incorrecto.

**Recomendación:**
- Si no hay payrolls en el semestre, el cálculo debe fallar explícitamente
- No debe caer a períodos anteriores

### **PROBLEMA 3: Cálculo de días para nuevos empleados**

El código hace:
```typescript
const effectiveStartDate = new Date(startDate);
effectiveStartDate.setDate(effectiveStartDate.getDate() + 1);
```

- **Total: 194 días de antigüedad en la fecha del cálculo (30/06/2026)**
- Para SAC semestre 1, solo cuenta enero a junio = ~181 días

⚠️ El cálculo parece estar sumando todos los días hasta el fin del semestre, lo cual es correcto para SAC.
